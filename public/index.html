Guide

Before You Start

  1. An IPv4 address for the service you want to expose.
  2. A quick way to test DNS propagation, such as ping or a health endpoint.
  3. A management password (minimum eight characters recommended) to confirm future edits.

How It Works

1. Check availability

Enter your desired subdomain and select Check. The tool validates formatting and queries availability in real time. Allowed characters are lowercase letters, numbers, and single hyphens between characters.

2. Create with IP and password

Provide the IPv4 address and a password. The platform writes the DNS A record immediately and queues it for global propagation, which typically completes within one to three minutes.

3. Manage with confidence

Authenticate with the same password whenever you need to update or remove the record. Every change is logged for auditing, and you can rotate IP targets as often as required.

Default TTL is 300 seconds. Edge locations sync through Cloudflare, so most visitors see updates nearly instantly. Contact the team if you require custom TTL or bulk operations.
